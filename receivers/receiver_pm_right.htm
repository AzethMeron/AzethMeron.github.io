PYRAMID (PM)
RECEIVER

The PM Receiver has a dual purpose. It's used to set or check the spell reward for defeating the guardians of a pyramid but is also a command that can be used to help attract the AI to new objects, or conversely to keep the AI from visiting them.

 These are the objects listed in the WoG Objects section of this help file.

The AI will try to visit an object (Pyramid) if:
 1. It has enough power to beat guards.
 2. It has enough wisdom to learn a spell there.
 3. If this object is NOT visited by any player.
 4. If this object is NOT visited by this player.
We cannot control 2 first conditions but we can set 2 others.

!!PMx/y/z:XXXX; Pyramid or any New Object.
You can use V command to "attract" AI to a New Objects (and Sphinx also).
You can set this bit and wait until the AI comes.

OPTIONS

P#1/$1; Set/check/get a "visited by player" flag. 
   #1 - Colour of the visitor (See Format E1).
   $1 - State of visit
         0 - Not visited   
         1 - Visited, 
S$; Set/check/get a bonus spell (AI will get it if it wins the "Pyramid" battle).
   $ - Spell number (Format SP).
V$; Set/check/get a "not visited" flag. 
   0 - Visited
   1 - Not visited,

Comments:
You can use the V command to "attract" the AI to New Objects (and Sphinx also).
You can set this bit and wait until the AI comes. If you catch it by !!OB receiver, disable the object, reset V bit and set P bit, you can make AI not have a battle and get a spell. So to attract the AI set the V flag. To free the AI, reset V and set P flag.
