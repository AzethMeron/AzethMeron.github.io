LODs MANAGEMENT (LD)
RECEIVER

Used for commands that load/unload custom LOD files, containing various resources such as graphics and animations.

Loaded LODs are NOT stored in saved game, instead there is a list of them. They are automatically loaded from given directory.

Note: LOD will be loaded EVEN if it is already loaded, thus be careful with using LD within repeatable triggers. What about multiplayer though?

!!LD:XXXX; Manage LOD files

OPTIONS

L?$1/$2/$3; Load a custom LOD file
    $1 - a variable to store index of the loaded LOD file
    $2 - a location folder for the file like in UN:J8
    $3 - an index of Z variable that stores a name of the LOD file
Comment:
Loaded LOD files will be unloaded if you start a new game or load a save. The list of loaded LOD files is stored in the saved game and thus all of them will be uploaded again if you load this save.
$1 returns >=0 if loading successful, otherwise it is -1
$2 there is new path option, for Mods folder: 11
$3 may be local, global or static Z variable
T?$1/$2/$3; Load a custom LOD temporarily
    $1 - a variable to store index of the loaded LOD file
    $2 - a location folder for the file like in UN:J9
    $3 - an index of Z variable that stores a name of the LOD file
Comment:
Same as LD:L except loaded files are NOT listed in saved game.
U$; Unload a custom LOD file
    $ - a variable storing an index of the loaded LOD file
Comment:
Naturally, index must be >=0


There is no need for filelist.txt in custom lods.

You can use a several load/unload sequence in one game. Say you can use a different environment for the adventure map, dialogs or anything else.

Note though that standard TXT files (like crtraits.txt and so) in game are parsed at game start. so even if you modify it, it does not affect the game unless you use "Data/h3wog.lod", "Data/h3std.lod" or "Data/h3custom.lod". Those are automatically loaded at game setup.

Example. 
!#VRz35:S^my.lod^;
!#LD:L?v3/2/35;
This will load my.lod file from MAPS folder.
